May 2, 2016

Updating project to GitHub

  • I've given up on the several step process of updating my github repo - my guess is that it is multi step to keep people from accidentally messing up production code (guessing people care more about their projects than I do). To be clearer though, here is the process I'm going through:
    • Writing code in Atom and saving the file to a folder on my computer
    • Move that file into a file on my machine "jjhonson.github.io" which is cloned(?) on GitHub
    • When I move or change a file it shows up in my GitHub desktop in "changed" column
    • Then I "Commit to master" with some statement which will likely not make sense to me in the future (I assume people working together would want that message to be really descriptive so others could see what has changed)
    • Then - I guess as one last fail-safe - I hit the Sync button.
  • Now that I've figured this out, it doesn't come across as difficult at all but it took me some time to figure it out.
  • There is a command to "Open on GitHub" - but I'm not sure what that does yet and if it skips any of these steps.
  • Beginning to work with CSS (have 5 exercises up on the page) and that's going really easy.
  • Atom makes it pretty easy because I can frequently just start typing things I want to do and it will suggest the right tags or code. The live previews help a ton too.
  • Everything is still pretty clunky but its coming along. The photo grid project is cool looking and significantly less embarassing than most of the other ones I've done.
  • One thing driving me crazy - I have no idea how to move the stupid .gif on on my index page over to the middle of the page. I can style img tags but for some reason none of the styling seems to work on this. I was able to put an ugly green border around it - so there's that.
